OSC Touch Editor

Help & Getting Started Guide

OSC Touch Editor is a browser-based control surface designer. Build custom touch interfaces with sliders, knobs, buttons, XY pads, and more — then send and receive Open Sound Control (OSC) messages to control any compatible software: Max/MSP, Pure Data, TouchDesigner, REAPER, Resolume, QLC+, and hundreds of others.

What is OSC Touch Editor?

OSC (Open Sound Control) is a network protocol used by audio, video, and lighting software to communicate in real time. OSC Touch Editor gives you a drag-and-drop canvas where you place control widgets, assign each one an OSC address, and then use your touch screen or mouse to send values to any software that speaks OSC.

Because browsers cannot send raw UDP packets (which OSC traditionally uses), this tool communicates via WebSocket. A small bridge script running on your local machine (or on a server) receives the WebSocket messages and forwards them as UDP OSC to your target software.

No installation needed for the editor. Open index.html in any modern browser — Chrome, Edge, or Safari. Firefox works too, but Chrome gives the best touch performance on tablets.

Quick Start (5 minutes)

  1. Open the editor. Load index.html in Chrome. If you're hosting it on GitHub Pages, just navigate to your page URL.
  2. Add a widget. Click Add → H-Slider in the toolbar. A slider appears in the centre of the canvas. It's selected automatically.
  3. Configure it. In the Properties panel (right side), set the OSC Address to something like /filter/cutoff and set Min/Max to 0–127.
  4. Start the bridge. Download and run the Node.js bridge script below. It listens for WebSocket connections and forwards to your target software.
  5. Connect. Click OSC → Bridge, enter ws://localhost:8080, click Save & Connect. The status dot turns green.
  6. Switch to Perform mode. Press Tab or click PERFORM in the toolbar. Now drag the slider — OSC messages are sent in real time.

Why do I need a bridge?

Web browsers run in a sandboxed environment. They can open WebSocket connections but cannot send raw UDP packets — which is how OSC is traditionally transmitted. The bridge is a tiny Node.js script that sits in the middle: 

Browser (WebSocket) ──→ Bridge script ──→ Your software (UDP OSC)

You run the bridge on the same machine as your target software (or on any machine on the same network). The bridge script supports three message formats: JSON, osc.js extended JSON, and plain text.

Tip: Open Stage Control also includes a built-in WebSocket server that this editor can connect to directly — no separate bridge needed if you're already using OSC.

Node.js Bridge Script

The bridge script below handles everything: WebSocket server, OSC UDP send/receive, bidirectional feedback, and reconnection. It requires Node.js 16+ and two npm packages.

Installation

# Install Node.js from https://nodejs.org if you haven't already

# Create a folder and install dependencies
mkdir osc-bridge && cd osc-bridge
npm init -y
npm install ws node-osc

Usage

# Basic — listens on WS port 8080, sends OSC to localhost:57120
node osc-bridge.js

# Custom ports and target
node osc-bridge.js --ws 9000 --host 192.168.1.50 --port 57110 --listen 57121

# With verbose logging
node osc-bridge.js --verbose

Pages

Pages let you build multiple independent canvases within one layout file. Each page has its own set of widgets and its own canvas background.

Managing pages

  • Add — click ＋ at the right of the tab bar.
  • Rename — double-click any tab to edit the name inline.
  • Delete — hover a tab and click ✕, or right-click → Delete.
  • Reorder — drag tabs left and right to change order.
  • Duplicate — right-click (or long-press) a tab → Duplicate. All widgets are copied with new IDs.
  • Move left/right — available in the right-click page settings popover.

OSC address per page

Each page has a configurable OSC address in its settings (right-click the tab). When Send on switch is enabled, switching to that page sends OSC_ADDRESS 1 — useful for telling your software which page is active.

Canvas Backgrounds

Click 🖼 Canvas BG in the toolbar to open the canvas background panel. Each page has its own independent background. Supported media:

  • Image — any URL (https://…/image.jpg) or local file (embedded as base64, persists in saves).
  • Video (MP4) — any URL or local file. Local files use a browser blob URL (session-only — needs reselecting after reload). URL-based videos persist fully.

Controls: Size (Cover/Contain/Stretch/Actual), Position (9-point grid), Repeat (images only), Opacity.

Images & Video in Widgets

Any widget (except Label/Text) can have a background image or video via the Background Media section in its properties panel. The media renders behind the control graphics — the slider fill and knob arc sit on top of it.

The Video widget is a standalone playback element. Add it from the toolbar and configure the source URL or pick a local file in its properties. It plays back with full Autoplay/Loop/Muted/Controls options, independent of OSC.

OSC Basics

Open Sound Control is a message-passing protocol over UDP (or TCP). Every message has an address (like a URL path) and one or more arguments (values). 

# A typical OSC message
/mixer/ch1/volume  0.75

# Multiple arguments
/xy/pad1  0.3  0.8

# Trigger (no value needed)
/transport/play  1

This editor sends a single numeric argument per widget. The XY pad sends two messages — one for X (/xy/x) and one for Y (/xy/y).

Sending OSC

Every interactive widget sends its value as an OSC message when touched in Perform mode. Set the address in the widget's OSC Address property.

Message formats

Choose your format in the OSC Bridge settings modal:

FormatExampleCompatible with
JSON{"address":"/vol","args":[0.5]}Most bridge scripts, custom code
osc.js{"address":"/vol","args":[{"type":"f","value":0.5}]}osc.js, Node.js bridges
Plain text/vol 0.5Simple bridges, telnet testing

Value ranges

Set Min/Max per widget using the Preset dropdown. The value sent is always within the configured range. Use 0–127 for MIDI CC values, 0–1 for normalised parameters, -1–1 for bipolar controls like pan.

Receiving OSC

The editor can receive OSC feedback from your software and update widget displays in real time — useful for metering, state synchronisation, and showing values that change independently of user interaction.

Setting up receive

  • Make sure your bridge script is running with a --listen port.
  • Configure your software to send OSC to that port on localhost.
  • The bridge forwards messages back to the browser via WebSocket.
  • Any widget whose OSC address matches receives the value and updates its display.

Receive-only widgets

Check Receive only in a widget's properties to make it display incoming values without reacting to touch. Use this for VU meters, playback position displays, or any feedback-only control. A green RX ONLY badge appears in edit mode.

XY pad receive

The XY pad has separate OSC addresses for X and Y. You can drive X and Y from different addresses — for example /xy/pan and /xy/depth.

OSC Monitor

Click ⎍ Monitor in the OSC toolbar group (or press M) to open the floating monitor window. It shows all inbound and outbound OSC messages in real time.

  • Filter pills — show All, ▶ Out only, or ◀ In only.
  • P — pause/resume logging.
  • ⌫ — clear the log.
  • ─ — minimise to title bar.
  • Drag the title bar to reposition. Resize from the bottom-right grip.

The monitor stores up to 500 messages and auto-scrolls. Scroll up to browse history — auto-scroll resumes when you return to the bottom.

Page Switching via OSC

Pages can be triggered by incoming OSC messages. Three matching methods are supported:

MethodExampleHow it works
Exact address/lighting 1Matches a page whose OSC address is /lighting
Index/page/2 1Switches to the 2nd page (1-based)
Name/page/Audio 1Matches a page named "Audio" (case-insensitive)
Value 0 is ignored — send value 1 (or any non-zero number) to trigger a page switch. This prevents toggle-style footswitches from accidentally switching away.

Kiosk Mode

Kiosk mode removes the entire editor UI and locks the layout into Perform mode. It's designed for dedicated touch screen installations.

Activating kiosk mode

Add ?kiosk to any URL:

https://yourdomain.com/index.html?kiosk
https://yourdomain.com/index.html?kiosk&theme=stealth

Generating the kiosk URL

Click ⊞ Kiosk URL in the toolbar. The modal offers:

  • Plain URL — short, uses localStorage on the target device.
  • Embed layout — bakes the entire layout as base64 into the URL. Works on any device, any browser, with no prior setup. Large layouts with embedded images make longer URLs.
  • Theme override — bake a specific theme into the URL regardless of the device's saved preference.
Security note: Anyone with the kiosk URL can view and control the layout, but cannot access the editor. The editor is inaccessible unless the URL is changed to remove ?kiosk.

Save & Load

  • Autosave — the layout is automatically saved to localStorage every 5 seconds and on page unload. It's restored automatically on next load.
  • Save (↓) — downloads a .json file you can share or back up.
  • Load (↑) — imports a saved .json file, replacing the current layout.
Local files (images/videos): Images are embedded as base64 in the JSON and persist fully. Videos selected from a local file use browser blob URLs which are session-only — they won't survive a page reload. Use URLs for video sources in production deployments.

Software Integration

Max/MSP

Use the udpreceive object to receive OSC, and udpsend to send feedback.

--- Receive from bridge (bridge listens on 57120 and forwards here) ---
[udpreceive 57120] → [oscparse] → route /mixer/ch1/volume

--- Send feedback back to bridge ---
[pack /mixer/ch1/volume 0.5] → [oscformat] → [udpsend 127.0.0.1 57121]

Or use the Node4Max package for direct WebSocket integration without UDP.

Pure Data

Use the mrpeach externals for OSC support:

[netreceive -u 57120] → [unpackOSC] → [routeOSC /mixer/ch1/volume] → [print]

TouchDesigner

Add an OSC In DAT with port 57120. In the parameter table, match the address column to your widget addresses. For sending feedback, use OSC Out DAT pointed at localhost:57121 (your bridge's listen port).

REAPER

Install the ReaOSC plugin or use the built-in Web interface + OSC support. In REAPER → Preferences → Control Surfaces, add an OSC device on port 57120. Map parameters using the OSC pattern matching.

Resolume Avenue / Arena

Go to Preferences → OSC, enable OSC input on port 57120. Widget OSC addresses must match Resolume's parameter paths exactly, e.g. /composition/layers/1/video/opacity.

QLC+

In QLC+ → Inputs/Outputs, add an OSC plugin input on port 57120. Map widget addresses to virtual console buttons and faders via the input profile. Use 0–255 range for faders or 0/1 for toggles.
